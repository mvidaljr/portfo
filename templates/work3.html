003 : Advantages of Using GridSearch or RandomSearch in Your Prediction Algorithm

As described in other articles of mine, the main focus of a data scientist is to have excellent accuracy of his predictive machine and reduced numbers of false positives and false negatives in the confusion matrix.

The challenge is to choose the algorithm that would best adapt to the resolution of the problem being worked on. In this article we will focus on which procedures to use to increase accuracy by manipulating the hyperparameters of the selected algorithms.

All Machine Learning algorithms are parameterized, which means that you can tune the predictive model by adjusting the parameters. There are several ways to optimize the hyperparameters of the selected algorithm. GridSearch, RandomSeach, Bayesian Optimization with Gaussian Process, Hyperopt and Optuna are some of them. But we will only talk about the two most used in the job market, which are GridSearch and RandomSearch from the Sklearn library.

1) sklearn.model_selection.GridSearchCV

The GridSearch function is well known and used, however it was not made to be used with many parameters, as an excessive amount of information could crash your machine. GridSearch methodically performs combinations between all parameters of the selected algorithm.


















In the above example, we used GridSearch to optimize the hyperparameters of the classification algorithm called Logistic Regression (LogisticRegrassion) from the SKlearn library. According to the documentation, Logistic Regression accepts several hyperparameters, but in this example we will only manipulate the penalty (penalty) and the C parameter. The test hyperparameters were stored in a key variable called values_grid. To check the meaning of each hyperparameter, access the Sklearn library link below.

We can analyze that after training our prediction algorithm, and being tested with each hyperparameter value, it was found that the Logistic Regression had a better behavior with the parameters ( C = 0.001, penalty = ‘none’) resulting in an accuracy of 95.931 or 95.93%.

2) sklearn.model_selection.RandomizedSearchCV

RandomSearch is the most suitable because it randomly chooses the test parameters. Unlike GridSearch, RandomSearch can train with more parameters without the machine crashing. Because it is a function that randomly chooses the values, we must indicate how many times the selected algorithm must choose the parameters called the number of interactions.

We will use the same dataset from the previous example, but we will modify the prediction algorithm for Decision Tree Classification (DecisionTreeClassifier) from the SKlearn library.

If we were to use GridSearch in this example, it would train with all values stored in the value_grid variable. It will do a total of 30 iterations before passing the best parameter values.

We can analyze that RandomSearch determines the number of interactions that the algorithm makes, being informed by the interaction variable.























We can analyze that in these 14 interactions that we determined, RandomSearch determined that the best parameters were (max_depth = 4, max_features = 5). Finding an accuracy of 95.909 or 95.90%.

Conclusion

Despite the various ways of optimizing the algorithms used, GridSearch and RandomSearch will be the most used in the market today. For a data scientist, hyperparameter optimization is essential for a smooth functioning of your algorithm.
