001 : How to increase the accuracy of your Machine Learning project

We know that the duration of a project depends a lot on how well the data is formulated. Many say that it takes more than 80% of a project’s time to identify the problem and do the whole process of extracting, organizing and cleaning data. It’s a very time-consuming process, but one that can make a huge difference when it comes to making machine learning predictions. In this article we will show you three important functions on how to increase the accuracy of your predictions.

1. sklearn.preprocessing.OrdinalEncoder

OrdinalEnconder is a function from the SciKit Learn library where it works by transforming categorical columns into an array of integers. For example, in a column where we have 3 dog breeds (Pitbull, Doberman and Rottweiler), we can transform the values ​​of each breed into representative integers. In this case, the pitbull being represented by the value 1, the doberman by the value 2 and the rotwaller by the value 3. This method would prevent the creation of new binary columns. Below we have a column represented by seven colors.

Column before apply Ordinal Encoder


Column after apply Ordinal Encoder

This method is essential to assist in prediction models, since the machines work better with numerical values ​​than categorical values.

2. sklearn.preprocessing.StandardScaler

StandardScaler is a function from the Scikit Learn library where its purpose is to use the mean and standard deviation of the training data to make a calculation with smaller sample values close to 0.

Columns before apply Standard Scaler


Columns after apply Standard Scaler (Numbers near value 0)

Thus, it will not be necessary to predict a table using high values.

3. imblearn.over_sampling.SMOTE

The imblearn library provides us with a very interesting function called SMOTE. This function creates records of the positive class. In the “color” column of the table below, we have more records of white wines (represented by the numerical value 1.0) than red wines (represented by the numerical value 0.0). This imbalance can lead to poor performance in the training part of your algorithm.

We can see that we have three times more values of white wines (1.0) than red wines (0.0). SMOTE will do an even balancing and this will have to be done before separating training and testing variables. The image below shows the amount of white and red wines before (1554, 4771) and after (4771, 4771) applying SMOTE to the dataset.

Columns before apply Smote


Columns after apply Smote (Reduce inbalanced dataset)

After balancing the values, it will be possible to obtain better accuracy in the predictions. Now that we have a balanced amount for both white and red wines, we can train the machine confident that the results will be more accurate.

Conclusion

These three functions helped me a lot in Machine Learning prediction, which can enhance its accuracy. All three functions have hyper parameters that can be used to prepare your table before it is trained.
